# API_SPEC.md

## Overview
FramePro's backend (`apps/api`) exposes a RESTful JSON API built with FastAPI. This document defines conventions and the core endpoint surface. GraphQL is not used unless a specific use case demonstrates a clear benefit (see `GEMINI.md`).

Base URL (production): `https://api.framepro.com/v1`

Interactive docs are auto-generated by FastAPI at `/docs` (Swagger UI) and `/redoc`, and should be treated as the always-up-to-date source of truth for request/response schemas. This document describes structure and conventions; the OpenAPI schema is authoritative for field-level detail.

---

## Conventions

- All endpoints are versioned under `/v1`.
- All request/response bodies are JSON; `Content-Type: application/json`.
- Authentication via JWT bearer token: `Authorization: Bearer <token>`.
- Pagination: cursor-based for large collections — `?cursor=<token>&limit=20`.
- Errors follow a consistent envelope:

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "width must be greater than 0",
    "field": "width"
  }
}
```

- Timestamps are ISO 8601 UTC (`2026-07-21T14:30:00Z`).
- Monetary values are represented as integer minor units (cents) with an explicit currency field, never floats.

## Auth

| Method | Path | Description |
|---|---|---|
| POST | `/auth/register` | Create a consumer account |
| POST | `/auth/login` | Email/password login, returns JWT |
| POST | `/auth/refresh` | Refresh access token |
| POST | `/auth/logout` | Invalidate session |
| POST | `/auth/distributor/apply` | Submit distributor application |
| GET | `/auth/me` | Current authenticated user profile |

Roles: `consumer`, `designer`, `distributor`, `admin`. Enforced via dependency-injected role checks in FastAPI, not just UI-level hiding.

## AI Endpoints

| Method | Path | Description |
|---|---|---|
| POST | `/ai/frame-preview` | Submit photo + SKU, returns job ID |
| POST | `/ai/room-visualizer` | Submit room photo + artwork/frame selections, returns job ID |
| GET | `/ai/jobs/{job_id}` | Poll job status (`pending`, `processing`, `completed`, `failed`) and result URL |
| POST | `/ai/recommend` | Submit photo/room context, returns recommended frame attributes |
| POST | `/ai/search` | Natural language product search |
| POST | `/ai/chat` | Sales assistant chatbot (streaming response) |

**Example: Submit a room visualizer job**

Request:
```json
POST /v1/ai/room-visualizer
{
  "room_image_key": "r2://uploads/rooms/abc123.jpg",
  "placements": [
    { "artwork_image_key": "r2://uploads/art/def456.jpg", "frame_sku": "FP-2201-WAL", "width_cm": 60 }
  ]
}
```

Response:
```json
{
  "job_id": "job_9f8a7b6c",
  "status": "pending"
}
```

Poll:
```json
GET /v1/ai/jobs/job_9f8a7b6c
{
  "job_id": "job_9f8a7b6c",
  "status": "completed",
  "result_url": "https://cdn.framepro.com/results/job_9f8a7b6c.jpg"
}
```

AI endpoints are rate-limited per user and per IP to control cost (see `DEPLOYMENT.md` / Security section).

## Webhooks (outbound)

FramePro emits webhooks for integration with distributor/partner systems:

| Event | Payload summary |
|---|---|
| `order.created` | Order ID, line items, total |
| `order.status_changed` | Order ID, previous/new status |
| `quotation.signed` | Quotation ID, signer, timestamp |
| `ai.job.completed` | Job ID, result URL |

Webhook payloads are signed with an HMAC secret per subscriber (header: `X-FramePro-Signature`).

## Error Codes (common)

| Code | Meaning |
|---|---|
| `VALIDATION_ERROR` | Request body failed schema validation |
| `UNAUTHORIZED` | Missing/invalid auth token |
| `FORBIDDEN` | Authenticated but lacking required role |
| `NOT_FOUND` | Resource does not exist |
| `MOQ_VIOLATION` | Order quantity below minimum order quantity |
| `INVENTORY_INSUFFICIENT` | Requested quantity exceeds available stock |
| `AI_JOB_FAILED` | AI generation job failed (see job detail for reason) |
| `RATE_LIMITED` | Too many requests |

## Versioning & Deprecation

- Breaking changes require a new version prefix (`/v2`), not in-place changes to `/v1`.
- Deprecated fields are marked in the OpenAPI schema with `deprecated: true` and kept for at least one full release cycle before removal.

Related: `DATABASE.md` for underlying schema, `AI_PIPELINE.md` for what happens inside AI job processing.
